# 5G核心网架构概述

5G核心网（5G Core，简称5GC）是5G系统的控制中枢，负责管理接入认证、会话建立、策略控制、用户面数据转发等关键功能。与4G EPC（Evolved Packet Core）相比，5GC在架构设计上进行了根本性变革。

## 服务化架构（SBA）

5GC采用**服务化架构（Service-Based Architecture，SBA）**，这是3GPP在TS 23.501中定义的核心架构范式。在SBA中，每个网络功能（Network Function，NF）以**服务生产者**或**服务消费者**的角色工作，通过RESTful API风格的接口（基于HTTP/2）相互通信，而非4G时代的点对点参考点连接。

SBA的核心优势包括：

- **模块化**：各NF独立部署、独立扩缩容，适合云原生环境。
- **服务发现**：通过NRF（NF Repository Function）实现自动化的NF注册与发现。
- **灵活组合**：新功能可以通过消费已有NF的服务快速构建。

## 关键网络功能（NF）

5GC定义了以下主要网络功能：

| 网络功能 | 全称 | 核心职责 |
|---------|------|---------|
| **AMF** | Access and Mobility Management Function | 接入与移动性管理，连接RAN，处理注册、切换 |
| **SMF** | Session Management Function | PDU会话管理、UE IP地址分配、UPF选择与控制 |
| **UPF** | User Plane Function | 用户面数据包路由/转发、QoS执行、流量报告 |
| **PCF** | Policy Control Function | 策略与计费控制，下发PCC规则 |
| **UDM** | Unified Data Management | 统一数据管理，存储签约数据，生成鉴权向量 |
| **AUSF** | Authentication Server Function | 鉴权服务功能，支持5G-AKA和EAP-AKA' |
| **NRF** | NF Repository Function | NF注册与发现，维护可用NF实例目录 |
| **NSSF** | Network Slice Selection Function | 网络切片选择，为UE确定合适的切片 |
| **NEF** | Network Exposure Function | 将5GC能力安全地暴露给第三方应用 |
| **AF** | Application Function | 应用功能，与核心网交互以影响策略和路由 |

此外还有**UDR**（Unified Data Repository，统一数据仓库）、**SMSF**（SMS Function）、**5G-EIR**（设备身份寄存器）等辅助功能。

## 控制面与用户面分离（CUPS）

5GC原生实现了**控制面（CP）与用户面（UP）的完全分离**。控制面功能（AMF、SMF、PCF等）集中部署，用户面功能（UPF）则可以分布式下沉到网络边缘，靠近用户部署，从而降低传输时延、提升用户体验。

这一设计理念源自3GPP在Rel-14中为4G EPC引入的CUPS（Control and User Plane Separation）架构。CUPS将SGW和PGW分别拆分为控制面（SGW-C、PGW-C）和用户面（SGW-U、PGW-U）组件。5GC在此基础上更进一步，将SMF和UPF作为原生分离的独立网络功能，通过N4接口（基于PFCP协议）通信。

## 4G EPC到5GC的演进映射

| 4G EPC网元 | 5GC网络功能 | 说明 |
|-----------|------------|------|
| MME | AMF + SMF | MME的移动管理功能由AMF承担，会话管理功能由SMF承担 |
| SGW + PGW | UPF | 用户面网关功能统一由UPF承担 |
| SGW-C + PGW-C | SMF | CUPS拆分后的控制面功能归入SMF |
| HSS | UDM | 签约数据管理演进为统一数据管理 |
| PCRF | PCF | 策略与计费规则功能 |
| — | NRF | 5GC新增，提供服务发现能力 |
| — | NSSF | 5GC新增，支持网络切片选择 |
| — | NEF | 5GC新增，支持能力开放 |
